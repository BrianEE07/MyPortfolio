# v1.1.0

## Summary
This version added the local holdings import workflow while keeping `data/holdings.json` as the single canonical runtime source for the public app.

## Holdings Import Flow
- Added a local source normalization layer instead of broker API integration.
- Kept the canonical holdings schema as:
  - `symbol`
  - `shares`
  - `cost_basis`
- Added `portfolio_app/holdings_import.py` as the source-adapter boundary.
- Expanded `scripts/import_holdings.py` into a general import CLI.

## Supported Inputs
- Canonical CSV
- Canonical JSON
- Firstrade Account History / Transactions CSV
- Optional static HTML rebuild after import

## Safety And Repo Boundary
- Added fail-fast validation so invalid imports do not overwrite canonical holdings.
- Switched canonical JSON writes to an atomic persistence path.
- Moved local manual inputs under `imports/` to separate repo-tracked runtime data from local source files.
- Updated repo ignore rules for local secrets, exports, and certificates.
- Added `docs/holdings-import.md` to document the local update workflow.

## Verification
- Added focused `pytest` coverage for canonical CSV import, canonical JSON import, and invalid-input fail-fast behavior.
- Verified local import can still feed static export without changing the public runtime interface.
