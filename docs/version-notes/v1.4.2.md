# v1.4.2

## Summary

This release adds a live runtime metrics overlay for local Flask rendering. Refreshing the local app can now update portfolio-level performance cards from the latest market prices plus generated historical snapshots, without rewriting the public data files.

## Data Runtime

- Added reusable portfolio metric calculation helpers so generated builds and runtime overlay use the same formulas.
- Added a live snapshot overlay that combines the latest generated snapshot cash with current holdings market value.
- Recomputes `Portfolio YTD`, `IRR`, `CAGR`, `Current Drawdown`, and `Max Drawdown` during Flask rendering when all live holding prices are available.
- Recomputes `Sharpe`, `Beta`, `Alpha`, and S&P 500 comparison values when enough benchmark history is available; otherwise those fields keep the generated build values.
- Preserves the build pipeline boundary: page refresh does not write `data/holdings.json`, `data/portfolio_metrics.json`, or `data/portfolio_snapshots.json`.

## UI

- Keeps the roadmap dialog header and close button visible while scrolling the roadmap content.

## Notes

- Cash balance, realized P&L, holdings count, and official historical snapshots still require the data build flow because they depend on transaction inputs.
- GitHub Pages remains static. It receives updated values when the static site is rebuilt and deployed.
