# v1.1.6

## Summary
This version continued the `Stock Details` cleanup for mobile and split the consolidated implementation log into per-version notes.

## Stock Details Mobile Polish
- Reworked sortable header layout so info-enabled columns no longer forced both the help `?` and the sort indicator into the same right-edge area on narrow screens.
- Moved the sort indicator below the label block on the left side so all three toggle states keep a stable header height without overlapping the help icon.
- Shortened the market-value header from `持倉市值 / Position Value` to `市值 / Value`.
- Rebalanced table widths so the details grid felt less cramped on phones.
- Kept the `P&L` column in the normal scrolling table flow after review, instead of pinning it on the right edge.

## Symbol Outbound Links
- Added Yahoo Finance outbound links to each stock symbol.
- Introduced a polished reveal interaction:
  - desktop hover and focus reveal the outbound icon
  - touch devices reveal the icon on the first tap
  - the icon opens the external quote page in a new tab
- Kept only one symbol row in an explicitly revealed state at a time and closed it on outside click or tab switch.

## Header Subtitle Orientation Fix
- Limited stacked subtitle formatting to phone portrait layouts.
- Restored a single-line subtitle in phone landscape with tighter spacing so it behaves more like the eyebrow line instead of the portrait stack.
- Moved the mobile portrait header actions slightly lower so the update block and theme toggle align more cleanly with the `投資儀表板` line.

## Section Heading Scale
- Reduced the tab title and supporting sentence sizing to `80%` of the previous scale so section headings feel lighter relative to the table content.
- Removed the tab-level supporting subtitle lines entirely so each section now opens directly with the main bilingual heading.

## Header Spacing Follow-Up
- Nudged the stacked mobile header-actions spacing slightly further apart and moved the theme toggle a touch lower beneath `Last updated`.

## Version Log Split
- Converted `docs/v1-implementation-log.md` into an index page.
- Split the previous large changelog into per-version notes under `docs/version-notes/`.
