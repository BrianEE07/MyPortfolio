# v1.2.1

## Summary
This patch fixes the `Top 10 Holdings` chart instability that could appear after switching tabs or changing phone orientation in the in-app browser.

## Chart Reflow Fix
- Identified the root cause as stale `Chart.js` canvas sizing when the overview tab was hidden and then shown again, or when the viewport dimensions changed during orientation updates.
- Reworked the frontend chart relayout path so the holdings and Fear & Greed charts can be rebuilt instead of only resized when the viewport or tab visibility changes.
- Added chart resynchronization hooks for:
  - tab activation
  - viewport resize
  - orientation change
  - returning to the page
  - returning from a hidden/background state
- Tightened the holdings chart container sizing rules so the canvas stays constrained to the `Top 10 Holdings` panel during reflow.

## Overview Label Stability
- Reworked the overview stat labels so bilingual card titles render through structured Chinese and English spans instead of one long combined string.
- Switched phone and tablet compact layouts to a stable two-line label treatment, which prevents the eight overview stats from wrapping unpredictably during orientation changes.
- Extended compact English labels to the tighter responsive breakpoints for longer items such as `Unreal. P/L`, `Real. P/L`, `Port. YTD`, and `Sharpe`.
- Removed the duplicate-label risk by rendering only one English label node per overview stat and swapping between full and compact copy in frontend logic based on the active breakpoint.
