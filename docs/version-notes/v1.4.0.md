# v1.4.0

Date: 2026-05-05

This release starts the data foundation for historical portfolio analytics and connects the Holdings Overview to the generated portfolio-level data.

## Transaction-To-Public-Data Pipeline

- Added a local canonical transaction model for `BUY`, `SELL`, `DEPOSIT`, `WITHDRAWAL`, and `FEE`.
- Added `INTEREST` as internal cash income for Firstrade financial rows.
- Kept private transaction history under `private/transactions.json`, outside the repo.
- Added a generator that rebuilds public `data/holdings.json`, `data/portfolio_metrics.json`, and `data/portfolio_snapshots.json` from local private inputs.
- Routed Firstrade CSV generation through the canonical transaction engine so holdings, metrics, and snapshots are produced together.
- Preserved the existing holdings runtime path: the Flask app and static export still read `data/holdings.json` for positions.
- Added `scripts/build_portfolio_data.py` as the preferred data build command for Firstrade CSV and private transaction inputs.

## Public Snapshots

- Added `data/portfolio_snapshots.json` as the first portfolio-level daily summary output.
- Snapshots include holdings market value, portfolio cash, total portfolio value, invested cost basis, unrealized P/L, realized P/L, and net external cash flow.
- The first version intentionally avoids publishing raw transactions or per-symbol daily history.

## Metrics

- Extended generated portfolio metrics with `twr`, `irr`, `cagr`, `current_drawdown`, and `max_drawdown`.
- Kept existing realized P/L fields compatible with the current overview cards.
- Changed `Portfolio YTD`, `Sharpe`, `Alpha`, and `Beta` to read generated portfolio metrics instead of estimating from the current holdings list.
- Removed duplicate public metric names: `Portfolio YTD` displays `twr`, and `irr` is the single money-weighted return field.

## Overview UI

- Added `Total Value` and `Cash Balance` to the primary overview cards.
- Changed the wealth-progress subtitle to use total portfolio value instead of invested holdings value only.
- Expanded the secondary metric rail to eight portfolio-level metrics: `Portfolio YTD`, `IRR`, `CAGR`, `Alpha`, `Beta`, `Sharpe`, `Current Drawdown`, and `Max Drawdown`.
- Added tooltips for the newly exposed metrics and mobile landscape label handling for the denser metric rail.

## Roadmap

- Moved `Data Pipeline & Snapshot Foundation` into the completed roadmap UI.
- Updated the next roadmap focus toward `Investment Overview 2.0`, allocation, and multi-asset positions.

## Documentation And Tests

- Added focused tests for transaction validation, cash ledger behavior, average-cost holdings reconstruction, realized results, daily snapshots, and public file generation.
- Added `docs/data-foundation.md` to document the private input and public output boundary.
