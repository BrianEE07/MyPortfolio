# v1.1.2

## Summary
This version was the first major `Stock Details` refinement pass, adding client-side sorting and a denser, more readable table layout.

## Stock Details Sorting
- Added client-side sorting for:
  - `Symbol`
  - `Price`
  - `Avg Cost`
  - `Shares`
  - `Market Value`
  - `1Y Drawdown`
  - `Trailing P/E`
  - `Forward P/E`
  - `P&L`
- Kept `Trend` as a static, non-sortable column.
- Set the default table order to `Market Value` descending.
- Added a three-step sort cycle:
  - primary direction
  - reverse direction
  - reset to default `Market Value` descending

## Table And Tooltip Refinement
- Highlighted the top three rows only in the default market-value sort.
- Tightened table density and rebalanced column widths for desktop review.
- Added concise bilingual tooltip help to harder-to-read columns.
- Reworked sortable help headers so the `?` icon sat beside the Chinese label while the sort arrow stayed at the far right.
- Simplified the trend title and clarified `P&L` ordering and tooltip copy.
- Added content-hash query strings to `styles.css` and `app.js` export URLs for cache busting.

## Additional UI Cleanup
- Refined the theme toggle into the compact two-icon capsule.
- Moved `1Y Drawdown` immediately left of `Trend`.
- Renamed the value column from `市值 / Market Value` to `持倉市值 / Position Value` during this iteration.

## Verification
- Rebuilt the static site successfully with `python portfolio.py --output docs/index.html`.
- Verified the Flask app still served `GET /` and `GET /health` successfully.
