# v1.4.3

## Summary

This release adds a public lightweight watchlist below Stock Details and documents the recommended local-first portfolio update flow.

## Watchlist

- Added `data/watchlist.json` as a public symbol-only watchlist; array order is the default display order.
- Added a `觀察清單 / Watchlist` table below Stock Details.
- Watchlist rows use the existing market data cache for price, trailing P/E, forward P/E, one-year drawdown, and alert signals.
- Alert signals currently cover deep drawdowns, 250-day trend breaks, large 60-day moving-average gaps, and high valuation checks.
- Watchlist sorting uses the same three-step interaction pattern as Stock Details: sort, reverse, then return to the JSON default order with no active arrow.
- Missing watchlist data or missing external market data falls back gracefully.

## UI And Market Data Polish

- Adjusted the roadmap dialog so only the circular close button stays fixed while roadmap content scrolls.
- Fixed S&P 500 trailing P/E parsing so Multpl rows marked as estimates are included instead of falling back to the older non-estimated row.
- Renamed the S&P 500 valuation card to `估算本益比 / Est. Trailing P/E` because recent Multpl values are estimated trailing P/E data, not a fully reported actual value or forward P/E.
- Localized stock trend labels to `站上年線` and `跌破年線`.

## Data Flow

- Documented the local-first flow for broker CSV updates.
- Clarified that raw broker exports remain under ignored local paths such as `imports/`.
- Clarified that GitHub Actions deploys public generated output and should not receive private trading records.

## Roadmap

- Added v1.4.3 to the completed roadmap baseline as `Watchlist & Local Automation Flow`.
