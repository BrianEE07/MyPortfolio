# v1.0.1

## Summary
This version focused on scheduled GitHub Pages refreshes and the first deployment-oriented cleanup around the modularized portfolio app.

## Highlights
- Updated GitHub Pages automation to run one scheduled refresh per day.
- Kept `push` on `main` and `workflow_dispatch` available for manual updates.
- Recorded the Taipei-time schedule mapping and later adjustment.
- Confirmed no UI, holdings-schema, or runtime-behavior change was introduced by this version.

## Broader v1 Foundation Recorded In This Milestone
- Added canonical holdings storage in `data/holdings.json`.
- Reduced `portfolio.py` to a thin CLI and app entry wrapper.
- Split the application into the `portfolio_app/` package for configuration, holdings, market data, snapshot assembly, templates, static assets, and routing.
- Replaced the old one-page layout with the `Holdings Overview`, `Stock Details`, and `Market Pulse` tabs.
- Introduced the light-default theme, dark-mode toggle, and static export under `docs/`.

## Verification
- Python syntax checks passed for the modular application structure.
- `GET /` and `GET /health` returned `200`.
- Static export completed successfully with `python portfolio.py --output docs/index.html`.

## Notes
- External market-data providers were unavailable during part of the local verification environment, but fallback paths still allowed the UI and export flow to render successfully.
